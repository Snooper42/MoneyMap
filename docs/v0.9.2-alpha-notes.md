MoneyMap v0.9.1-alpha UI/UX Polish Notes
Build
Release stage: Pre-v1 alpha
Build: v0.9.1-alpha
Use the HTML file as the repo root `index.html`.
Main changes
Navigation and device support
Fixed the tablet navigation gap where the sidebar could disappear before the bottom navigation appeared.
Bottom navigation now appears through iPad/tablet widths up to 1180px.
Bottom navigation is fixed to five actions: Home, Review, Txns, Budget, More.
More menu works as a bottom sheet on iPhone and iPad.
More menu uses a three-column layout on tablets and a two-column/one-column layout on smaller phones.
View changes scroll back to the top on mobile and tablet to reduce disorientation.
Mobile polish
Reduced vertical spacing across cards, hero blocks, forms, and page headers.
Tightened iPhone card density while preserving 44px touch targets.
Added safer iPhone drawer behavior with sticky bottom actions.
Improved toast and storage-warning placement above the mobile nav.
Added very-small-phone breakpoint for narrow devices.
Added orientation and viewport-height handling for iOS Safari behavior.
Tablet polish
Added dedicated iPad/tablet breakpoint behavior.
Top bar, search, grids, and nav now adjust cleanly between iPhone and desktop sizes.
Metric grids remain two-column on tablet instead of collapsing too aggressively.
Desktop polish
Tightened page width, card spacing, chart surfaces, buttons, and tables.
Reduced oversized gaps while keeping the dashboard spacious.
Improved sticky top bar behavior.
Charts
Applied the credit-tracker-inspired chart surface treatment to all `.canvas-wrap` charts.
Added consistent radius, subtle gradient, and top highlight treatment.
Ensured chart canvases are focusable for keyboard accessibility.
Accessibility and friction reduction
Preserved safe touch target sizes.
Added reduced-motion handling.
Added print cleanup rules.
Improved focus and drawer behavior.
Ensured mobile More menu closes after selection.
