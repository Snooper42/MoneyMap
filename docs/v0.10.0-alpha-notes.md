# MoneyMap v0.10.0-alpha Notes

## Scope

Started the structural split from one large HTML file into separate assets.

## Files

- `index.html`: app shell and markup
- `src/css/styles.css`: extracted CSS from all inline style blocks
- `src/js/app.js`: extracted JavaScript from all inline script blocks

## Why this matters

The prior monolithic file mixed app UI, CSS, scripts, report templates, and patch layers. This made mobile/nav fixes risky because code could be inserted in the wrong place. Splitting CSS and JS creates safer boundaries.

## Important

This is intentionally not a redesign. Future steps should split `app.js` into feature files such as:

- `storage.js`
- `navigation.js`
- `accounts.js`
- `transactions.js`
- `budgets.js`
- `investments.js`
- `charts.js`

## QA checklist

- Open fresh with no local data
- Load demo data
- Test Home
- Test Accounts if present
- Test Review
- Test Transactions
- Test Budgets
- Test Investments
